Obtaining and Configuring the YouTube API Key

Obtaining the YouTube API key was the first step. Here's how we used it in our code:

var youtubeApiKey = 'YOUR_YOUTUBE_API_KEY';

We embedded this key in our searchYouTube function:

function searchYouTube(query, maxResults) { const searchUrl = `https://www.googleapis.com/youtube/v3/search?part=snippet&q=${encodeURIComponent(query)} trailer&key=${youtubeApiKey}&maxResults=${maxResults}&type=video`; // API call logic... }

This function forms the backbone of our YouTube integration. However, we encountered API usage limitations. Ideally, we would implement a caching strategy to minimize redundant API calls, but this is an area we haven't explored yet.

Embedding the Video Player in the <aside> Element

We faced challenges embedding the video player. Initially, we tried integrating it within individual media cards, but sizing issues and YouTube's default UI made this impractical. Here's the code snippet where we settled on placing the player in the <aside> element using flexbox:

<aside> <div id="videoContainer"> <!-- Video iframe is dynamically inserted here --> </div> </aside>

We had to abandon our initial plan due to these constraints, opting for a more stable solution with the <aside> element.

Auto-Scroll Feature and 'Watch Trailer' Button

The 'Watch Trailer' button triggers an auto-scroll feature, which, admittedly, is still a bit janky. Here's the relevant code:

$('html, body').animate({ scrollTop: $("aside").offset().top }, 1000);

This snippet, triggered when a trailer is played, animates the page scroll to the <aside> element. It's functional, but there's definitely room for improvement in terms of smoothness and responsiveness.

The button itself is crucial in fetching the correct trailer:

$('.watch-trailer-button').on('click', function () { let movieTitle = $(this).data('movie-title'); searchYouTube(movieTitle, 1) .then(data => showTrailerPopup(data.videoId, data.startTime)) .fail(error => console.error(error)); });

This function uses the movie title, obtained from the OMDb API data displayed in the DOM, to fetch the trailer from YouTube.
